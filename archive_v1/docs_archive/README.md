# محرك HUSSAIN (Hybrid Engine for Semantic Islamic Navigation)

يُعد هذا المشروع قفزة نوعية في تمثيل واسترجاع المعرفة الإسلامية والفلسفية المستمدة من "المنظومة المعرفية لمفاهيم السيد حسين بدر الدين الحوثي". 
لقد تطور المشروع من مجرد مصفوفة ملفات بيانات (JSON) إلى **محرك بحث هجين متكامل** يربط مفاهيم الأنطولوجيا الفلسفية بجذور القرآن الكريم ولآياته بدقة دلالية متناهية.

## 🌟 الميزات الحالية (Core Features)

1. **الأنطولوجيا الموحدة (Unified Ontology):**
   تم دمج كافة المتفرقات والدروس في شبكة دلالية واحدة (`unified_ontology.ttl`) لبناء الرسوم المعرفية (Knowledge Graphs).
2. **محرك البحث المزدوج (Self-Routing Hybrid Search):**
   - استرجاع صلب (Hard Retrieval): جلب الآيات المرتبطة بالمفاهيم بشكل قطعي عبر خريطة `concept_ayah_mapping.json`.
   - استرجاع دلالي مرن (Semantic Fallback): استخراج الآيات القرآنية الأقرب لروح المفهوم الفلسفي باستخدام تقنيات المتجهات (Vector Embeddings) في حال غياب الاقتباس المباشر.
3. **التكامل اللغوي (Lexical Roots):**
   استخلاص الجذور الصرفية واللغوية للآيات المعروضة باستخدام قاعدة بيانات `SQLite` لتعزيز عمق الاستيعاب القرآني.

## 📂 هيكلية المشروع (Directory Structure)

- `hybrid_search.py` : العصب الأساسي للمحرك المُبرمج بـ Python.
- `quran_roots_dual_v2.sqlite` : قاعدة البيانات العلائقية للقرآن وجذوره.
- `unified_ontology.ttl` : الروافع والروابط الأنطولوجية الشاملة.
- `concept_ayah_mapping.json` : خريطة القاموس الثابت السريع للفهرسة.
- `project_documentation.md`: وثيقة الشرح المعماري التفصيلي لدورة عمل المحرك.
- `requirements.txt` : ملف الاعتمادات (Dependencies).
- `archive_v1/` : مجلد الأرشيف الحاوي للسكربتات التجريبية وملفات JSON السابقة.

## ⚙️ متطلبات التشغيل والتثبيت

تأكد من توافر بيئة `Python 3.9+`، ثم قم بتثبيت الاعتمادات المعيارية للمحرك:
```bash
pip install -r requirements.txt
```

لتشغيل المحرك (الفهرسة الآلية + البحث):
```bash
python hybrid_search.py
```
*(ملاحظة: في التشغيل الأول، سيقوم المحرك بتحميل نموذج النماذج اللغوية (BERT) وبناء قاعدة البيانات المتجهة ChromaDB للقرآن الكريم، وهو ما قد يستغرق بعض الوقت بناءً على قدرات الجهاز).*
