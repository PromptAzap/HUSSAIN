# وثيقة مشروع محرك HUSSAIN (الهجين)

## 📌 نبذة عامة
مشروع **HUSSAIN (Hybrid Engine for Semantic Islamic Navigation)** هو محرك بحث متقدم مصمم للربط بين "الأنطولوجيا والمفاهيم الفلسفية" (المنتزعة من دروس ومفاهيم السيد حسين) وبين "النص القرآني وجذوره اللغوية". يوفر المحرك إمكانيات غير مسبوقة لاستخراج الآيات القرآنية بدقة عالية بناءً على الدلالة الفلسفية العميقة للمعاني.

## 🏗️ الهيكلية التقنية (Architecture)
يعتمد المحرك على بنية هجينة تجمع بين تقنيات الذكاء الاصطناعي للاستخراج الدلالي (Semantic Retrieval) وتقنيات القواعد العلائقية للاستخراج اللغوي (Lexical Retrieval):

1. **الاسترجاع الدلالي (Vector Database):**
   - المحرك المعالِج: `sentence-transformers` باستخدام النموذج المعتمد `asafaya/bert-base-arabic` لتمثيلات الكلمات العربية الدقيقة.
   - قاعدة المتجهات: `ChromaDB`.
   - المجموعات (Collections):
     - `concepts_collection`: تخزن آلاف المفاهيم المستخلصة واسمها وتعريفها كمتجهات.
     - `quran_collection`: تخزن القرآن العظيم كاملاً (6236 آية) للرجوع الدلالي.

2. **الاسترجاع العلائقي اللغوي (Relational Database):**
   - مزود البيانات: قاعدة `SQLite` المشذبة `quran_roots_dual_v2.sqlite`.
   - تتم تغذية المحرك بالنصوص المجردة (text_plain) وتستخرج الواجهات الآيات بالرسم (text_uthmani).
   - توفر الجذور الصرفية والتحليل اللغوي لكل كلمة في كل آية.

3. **جسر الربط (The Knowledge Graph Bridge):**
   - تم بناء معجم مطابقة `concept_ayah_mapping.json` (Hard-Links) لربط المفهوم باقتباسه المرجعي، في حال تحقق شرط التشابه الدقيق (>4 كلمات).

## 🚀 دورة حياة البحث في المحرك (Query Lifecycle)
عند استعلام المستخدم عن مفهوم، يتخذ المحرك مسار التوجيه الذاتي (Self-Routing):
1. **التعرف المفاهيمي:** يترجم كلمة المستخدم إلى أقرب "تعريف فلسفي وأنطولوجي" من المعجم.
2. **الفحص الثابت:** إذا وجد أن هذا المفهوم يعتمد مباشرة على مرجعية لآية في الأنطولوجيا، يستقيها فوراً ويُظهر جذورها اللغوية.
3. **الذكاء الاحتياطي (Semantic Fallback):** إذا لم تتوفر مقولة تأسيسية قرآنية للمفهوم، يقوم المحرك بدمج اسم المفهوم مع تعريفه لإنتاج "سياق قوي"، ويقذف به في Vector DB للقرآن الكريم، فيستخرج الآية الأقرب لروح المفهوم ومعناه بدقة عالية.

## 📂 هيكلية الملفات الحالية (The Core)
الآن وبعد مرحلة الأرشفة وتنظيف المسار، هذه هي نواة المشروع الصلبة:
- `hybrid_search.py` : العصب الأساسي للمشروع وبرمجية الاسترجاع والتضمين.
- `quran_roots_dual_v2.sqlite` : مخزن المعرفة اللغوي للقرآن والجذور.
- `unified_ontology.ttl` : الروافع والروابط الأنطولوجية الشاملة.
- `concept_ayah_mapping.json` : خريطة القاموس الثابت السريع.
- `requirements.txt` : ملف اعتمادات البنية التحتية لتشغيل المشروع أو ترقيته لخادم واستضافة.
- `archive_v1/` : المجلد الأرشيفي الحاوي لملفات الـ JSON المفككة وبرمجيات الاختبار وبناء الجسور التجريبية.

## 🔮 الخطوات المستقبلية المقترحة
- **واجهة برمجة التطبيقات (API):** تغليف المحرك عبر `FastAPI` لتقديم مسارات مثل `POST /hybrid-search`.
- **ترقية قاعدة البيانات:** دمج `SQLite` و `ChromaDB` في قاعدة واحدة مستقبلاً عبر إضافة دعم `pgvector` في خادم PostgreSQL لتسهيل الصيانة.
- **تطوير واجهات المستخدم (UI Engine):** سواء باختيار React أو Next.js ليحظى المستخدم بتجربة بصرية فريدة مع الرسوميات والتفاعل الديناميكي.
