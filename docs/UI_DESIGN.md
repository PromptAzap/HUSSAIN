# 🎨 وثيقة التصميم الشاملة — منظومة HUSSAIN
**من منظور UI/UX + مطور تطبيقات تفاعلية**

---

## المقدمة: هوية المشروع من منظور التصميم

مشروع HUSSAIN ليس مجرد محرك بحث — هو **تجربة معرفية روحية وفكرية**. يتعامل مع تراث فكري عميق يمزج بين:
- الفكر القرآني والجذور اللغوية
- المنهجية التحليلية الفلسفية
- الإرث العلمي الإسلامي

هذا يفرض على المصمم أن يُترجم هذه الأبعاد إلى **تجربة مستخدم هادئة، عميقة، وذات وقار** — بعيداً عن الصخب البصري.

---

## القسم الأول: مبادئ التصميم الأساسية (Design Principles)

### المبدأ الأول: **الصمت البصري** (Visual Silence)

> "المعرفة الحقيقية تنكشف في الهدوء"

التصميم يجب أن يُزيل كل ما هو غير ضروري. كل عنصر بصري يجب أن **يخدم المحتوى**، لا أن ينافسه.

- ❌ لا إعلانات، لا تشتيت، لا ألوان صارخة
- ✅ مساحات بيضاء واسعة (White Space كفلسفة وليس تنسيق)
- ✅ نسبة نص إلى واجهة عالية — المحتوى هو البطل

---

### المبدأ الثاني: **الطبقات الدلالية** (Semantic Depth)

الواجهة يجب أن تعكس **عمق المحتوى نفسه** — كل نقرة تكشف طبقة أعمق:

```
السطح    → النص (الآية / الفقرة)
الطبقة 1 → المفهوم المرتبط
الطبقة 2 → الجذر اللغوي والسلسلة المفاهيمية
الطبقة 3 → الشبكة المعرفية الكاملة
```

المستخدم يبدأ بسيطاً ويغوص عميقاً — **دون أن يشعر بالتعقيد**.

---

### المبدأ الثالث: **السياق دائماً حاضر** (Always Contextual)

لا نتيجة بحث معزولة. كل نتيجة تظهر مع:
- من أين جاءت؟ (السلسلة / الدرس)
- ماذا سبقها وما الذي يليها؟ (السياق الفقري)
- ما صلتها بالقرآن؟ (الربط الدلالي)

---

### المبدأ الرابع: **ثنائية اللغة كهوية** (Bilingualism as Identity)

العربية **ليست ترجمة** — هي اللغة الأم للمشروع. التصميم يجب أن:
- يبدأ من **اليمين إلى اليسار** كفلسفة تصميم
- يستخدم الخط العربي كعنصر جمالي أساسي
- لا يُعامل RTL كـ "إضافة" بل كـ "الأصل"

---

### المبدأ الخامس: **التدرج المعرفي** (Knowledge Gradient)

من المبتدئ إلى الباحث المتخصص — الواجهة تتكيف:

| مستوى المستخدم | تجربة مقترحة |
|---|---|
| مبتدئ | بحث بسيط + نتائج موسّعة |
| متوسط | تصفية + استكشاف مرئي |
| متخصص | واجهة API + استعلامات متقدمة |

### 📱 التطبيق الأول: **HUSSAIN Scholar** — محرك البحث الرئيسي
**النوع:** Web App (PWA)  
**المستخدم المستهدف:** الباحث، الطالب، المثقف

#### الفكرة
واجهة بحث ذكية **أشبه بـ Perplexity.ai** لكن مخصصة تماماً للتراث الحوثي والقرآن. المستخدم يكتب سؤالاً بلغة طبيعية، والنظام يُجيب من مصادر المشروع.

#### شاشات رئيسية

**① الشاشة الرئيسية — Search as a Soul**
```
┌─────────────────────────────────────────────┐
│   ✦  منظومة HUSSAIN المعرفية               │
│                                             │
│   ┌─────────────────────────────────────┐   │
│   │  🔍  ابحث في التراث المعرفي...     │   │
│   └─────────────────────────────────────┘   │
│                                             │
│   اقتراحات:  #الإنسان  #الكفر  #القرآن    │
│                                             │
│   ── آخر البحوث ──────────────────────     │
│   • معنى الخسارة الحقيقية                  │
│   • الربط بين العقل والوحي                 │
└─────────────────────────────────────────────┘
```

**② صفحة النتائج — The Three Layers**
```
┌──────────────────┬──────────────────────────┐
│  SIDEBAR         │  MAIN RESULT             │
│                  │                          │

│  🔬 الأنطولوجيا  │  │  الخسارة الحقيقية  │  │
│  📚 الدروس       │  └────────────────────┘  │
│  📕 القرآن       │                          │

│  🔗 المفاهيم     │  ┌────────────────────┐  │
│  المرتبطة:       │  │ ﴿وَالْعَصْرِ...﴾   │  │
```

**UX المميز:**
- **"الخيط الذهبي"** — خط بصري يربط المفهوم → الآية → الدرس
- **وضع التركيز** — إخفاء كل شيء إلا النص القرآني عند القراءة
- **مشاركة النتيجة** — توليد صورة جميلة للمشاركة على السوشيال

### 🗺️ التطبيق الثاني: **المخيلة المعرفية** — Knowledge Graph Explorer
**النوع:** Interactive Web Visualization  
**المستخدم المستهدف:** الباحث والمفكر

#### الفكرة
**خريطة مرئية تفاعلية** للأنطولوجيا — كل مفهوم عقدة، والعلاقات بينها خطوط مضيئة. أشبه بـ **Obsidian Graph View** لكن بهوية إسلامية.

#### التجربة البصرية
```
        [الإيمان] ── [العمل]
           │              │
        [الوحي]        [التكليف]
           │              │
        [التفقه] ─── [الإسلام]
           │
      [الجهاد] ── [المقاومة]
```
- كل عقدة **"تتنفس"** — حركة نبضية هادئة
- مرور الماوس يُضيء الروابط
- النقر يفتح **"نافذة الكون"** — ملخص المفهوم + آياته + دروسه

**ميزة مميزة: "المسار الفكري"**
المستخدم ينقر مفهومَين ويرى الطريق الفكري بينهما:
```
الخوف → المراقبة → التقوى → الإخلاص → التحرر
```

### 📖 التطبيق الثالث: **رفيق الدرس** — Lecture Companion
**النوع:** Progressive Web App / Mobile  
**المستخدم المستهدف:** المتعلم اليومي

#### الفكرة
تطبيق قراءة يومية — **"درس في يوم"** — يقدم تجربة **القراءة المركّزة** للدروس مع ربط الآيات تلقائياً.

#### شاشات رئيسية

**① الشاشة الرئيسية — Daily Lesson**
```
┌─────────────────────────────────┐
│                                 │
│   الأحد، ٣٠ رمضان              │
│   درس اليوم                    │
│                                 │
│  ┌─────────────────────────┐   │
│  │  الدرس الثالث           │   │
│  │  سلسلة معرفة الله       │   │
│  │                         │   │
│  │  ████████░░░░  ٦٥%     │   │
│  │  مكتمل من قبل           │   │
│  └─────────────────────────┘   │
│                                 │
│   [ متابعة القراءة ]           │
│                                 │
│   ─── مميزة اليوم ───          │
│   ﴿وَمَا خَلَقْتُ الْجِنَّ...﴾  │
│   اكتشف ارتباطها بالدرس ←     │
└─────────────────────────────────┘
```

**② وضع القراءة — Immersive Reader**
- **خلفية بيج/كريم** دافئة — كورق الكتاب
- **حجم خط قابل للتعديل** مع ثلاثة أوضاع
- عند الوصول لفقرة تحتوي آية → **توهج ذهبي خفيف** حولها + نقرة لعرض الآية كاملة
- **"علّم هذه الفقرة"** — حفظ مع ملاحظة شخصية

**ميزة Killer: "الإضاءة الآية"**
كل مرة يمر المستخدم على نص قرآني، تظهر **طبقة شفافة** مع:
- نص الآية بالرسم العثماني
- الجذور اللغوية للكلمات المفتاحية
- المفاهيم المرتبطة

### 🤖 التطبيق الرابع: **المرشد الفكري** — AI-Powered Chat
**النوع:** Chat Interface (يعتمد على LLM + HUSSAIN كـ RAG)  
**المستخدم المستهدف:** الجميع

#### الفكرة
**محادثة ذكية** تعتمد على المشروع كـ Context (Retrieval-Augmented Generation). المستخدم يتحدث بلغة عادية، والنظام يجيب من مصادر HUSSAIN **مع الاستشهاد الدقيق**.

```
المستخدم: ما معنى الولاية في فكر السيد؟

المرشد: بناءً على ٥ دروس في سلسلة "معرفة الله":

الولاية عند السيد ليست مجرد سلطة...
[عرض النص الأصلي من الدرس الثاني←]

وترتبط بالآية الكريمة:
﴿إِنَّمَا وَلِيُّكُمُ اللَّهُ...﴾ [المائدة:٥٥]
[الجذور: و-ل-ي | المفاهيم: الطاعة، النصرة←]

─ المصادر المستخدمة:
• سلسلة معرفة الله / الدرس ٢ (٨٧% تطابق)
• سلسلة متفرقات / الدرس ١٥ (٧٤% تطابق)
```

**UX مميز:**
- كل إجابة تُظهر **"نسبة الثقة"** وعدد المصادر
- زر **"اقرأ المصدر كاملاً"** يفتح الدرس في نافذة منبثقة
- **وضع الاستجواب** — المستخدم يضع أفكاراً للمرشد ليناقشها

## القسم السادس: مبادئ UX خاصة بالمحتوى الديني/الفكري

### ١. احترام النص المقدس
- النص القرآني يُعرض دائماً بـ **خط مخصص واضح**
- لا يُقطع الآية بعناصر UI
- هامش وقار مرئي حول النصوص القرآنية

### ٢. عدم التحريف البصري
- لا تأثيرات تُشوّه النص العربي
- لا تدوير أو ضغط للخطوط
- المقروئية فوق الجمال في الأولوية

### ٣. التدرج لا الإغراق
- لا تعرض كل شيء دفعة واحدة
- **"الحجاب المعلوماتي"** — أظهر الجوهر، احجب التفاصيل حتى يُطلب

### ٤. إمكانية الوصول (Accessibility)
- دعم قارئات الشاشة بالكامل
- حجم خط قابل للتكبير دون كسر التصميم
- نسب تباين WCAA AA على الأقل
